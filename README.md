# CUE — Craft Universal Efficient Prompts

A prompt engineering skill that generates system prompts, agent prompts, user prompts, design descriptions, meta-prompts, and neutral descriptions using the **RACE-PEP** framework. Built for Claude Code, but applicable across LLMs.

---

## What CUE Stands For

**C · U · E** — three mental modes that map to how a prompt engineer actually thinks:

### **C — Capture**

Capture the essence. Distill what the user *actually needs* into a precise, actionable prompt. This is the chain-of-thought layer — the reasoning that happens before a single word is written.

- Parse intent from ambiguous language
- Identify the real deliverable behind surface requests
- Extract constraints the user forgot to mention
- Chain-of-thought reasoning: *what does this prompt need to produce, and why?*

> "Summarize X" → do the summary. "Write me a prompt to summarize X" → generate the prompt.

---

### **U — Untangle**

Untangle complexity. This is where the heavy cognitive lifting happens — a spectrum of operations that transform messy, underspecified requests into structured, solvable problems:

| Operation | What It Does |
|-----------|--------------|
| **Unifying** | Bringing disparate ideas into a coherent whole |
| **Untangling** | Working through complexity to find the signal |
| **Unpacking** | Breaking down assumptions or hidden details |
| **Unraveling** | Tracing a problem to its root causes |
| **Upscaling** | Expanding a concept to a broader abstraction |
| **Updating** | Revising a mental model with new information |
| **Upleveling** | Moving to a higher-level perspective |
| **Understanding** | Straightforward comprehension of the request |

> *Untangling* is the closest fit to the vibe of a thinking verb. It captures the core motion: taking something knotted and making it straight.

---

### **E — Engineer**

Engineer the prompt. Apply proven techniques, model-specific optimizations, and structural blueprints to produce a prompt that *works*:

- **RACE-PEP framework** — Role, Action, Context, Example, Expectation
- **Complexity calibration** — Simple (15 lines) vs. Medium (40-80) vs. Complex (80-150)
- **Token efficiency** — Static first, positive over negative, specific over vague
- **Model adaptations** — Claude (XML tags), GPT-4o (JSON mode), Gemini (hierarchical headings), and more

---

## The RACE-PEP Framework

Every prompt generated by CUE follows this structure:

| Step | Purpose | When to Use |
|------|---------|-------------|
| **Role** | Expert identity + tone | Open-ended/creative tasks |
| **Action** | Exact deliverable + method | Always |
| **Context** | Background knowledge | Domain-specific tasks |
| **Example** | Few-shot anchors | When format matters |
| **Expectation** | Format, length, structure | Always |

CUE supports six prompt archetypes, each with a dedicated blueprint:

| Type | Trigger Phrases | Use Case |
|------|----------------|----------|
| **System Prompt** | "system prompt", "AI identity" | Define agent behavior and boundaries |
| **Agent Prompt** | "agent that", "autonomous" | Multi-step workflows with tool usage |

| **Design Brief** | "design brief", "visual" | Creative direction and specifications |
| **Meta-Prompt** | "prompt template", "reusable" | Variable-driven prompt generators |
| **Description** | "describe", "explain" | Neutral, informative explanations |

## Complexity Calibration

CUE scales prompt complexity to match task difficulty:

| Level | Requirements | Lines | Extras |
|-------|-------------|-------|--------|
| **Simple** | 1-2 requirements | 15-30 | Output ONLY the prompt |
| **Medium** | 3-5 requirements | 40-80 | 2-3 examples |
| **Complex** | 6+ requirements | 80-150 | Workflow + error handling + state |

## Anti-Patterns Detected

CUE identifies and fixes these common prompt failures:

- Generic roles ("helpful assistant")
- ALL-CAPS instructions (models ignore them)
- Negative without positive reframing
- Vague constraints ("be concise" → "under 100 words")
- Multiple tasks in one prompt
- Exposing internal prompts/tools to end users

## Stress Tests

8 complexity dimensions tested with progressive injections:

| Dimension | What It Tests | Escalation |
|-----------|---------------|------------|
| **Reasoning Depth** | Multi-hop logic, counterfactuals | 2 steps → recursive self-verification |
| **Constraint Density** | Conflicting rules, trade-offs | 1 constraint → contradictory constraints |
| **Context Volume** | Signal-to-noise ratio | 1K tokens → 50K with buried critical info |
| **Output Structure** | Schema complexity, cross-references | JSON → self-validating schemas |
| **Ambiguity Resolution** | Vague inputs, implicit assumptions | Clear → actively misleading |
| **Meta-Cognition** | Self-correction, uncertainty | Direct → adversarial self-audit |
| **Domain Fusion** | Cross-disciplinary reasoning | Single → three+ domains |
| **Temporal/Stateful** | Sequences, state changes | Static → branching timelines |

### Injection Stack (Maximum Difficulty)

Progressive complexification applied on top of a seed prompt:

1. **Constraint Conflicts** — 5 contradictory rules requiring arbitration
2. **Multi-Format Output** — JSON + table + summary + audit trail
3. **Adversarial Input** — Misdirection, negation, competitor references
4. **Recursive Meta-Prompting** — Generate attack → defense → evaluate defense
5. **Stateful Memory Decay** — Turn history with rule overrides and forgetting

### Scoring Rubric

| Metric | Target |
|--------|--------|
| Constraint Adherence | 85%+ of hard constraints satisfied |
| Reasoning Transparency | Auditable decision process |
| Graceful Degradation | Fails with explanation, not silently |
| Token Efficiency | Under 2x baseline for complex tasks |
| Self-Awareness | Reports uncertainty when uncertain |

---

## Token Efficiency Rules

1. **Static first, dynamic last** — System instructions before user query (50-90% cache savings)
2. **Positive over negative** — "only use X" > "don't use Y"
3. **Specific over vague** — "3 bullets" > "be concise"
4. **One task per prompt** — Chain complex workflows

## Usage

CUE is designed as a Claude Code skill. Load it when you need to:

- Generate any prompt type (system, agent, user, design, meta, description)
- Optimize existing prompts for specific models
- Detect and fix prompt anti-patterns
- Calibrate complexity to task difficulty

### As a Claude Code Skill

```markdown
Use the cue-skill when the user wants to create, improve, or optimize any prompt type.
```

### Standalone

Read `references/blueprints.md` for the prompt structures. Apply RACE-PEP to any prompt you write.
